# 多题库目录 设计文档

## 背景

当前应用 `工程师题库` 是一个单一题库的练习 H5：所有题目都在 `questions.js` 的全局 `QUESTIONS` 数组中，进入应用直接看到「新建练习」按钮和历史记录。

需要新增一层「题库目录」，把现有题目归为 `中级工程师` 题库，并导入 `课后习题汇总.xlsx`(375 题，含章节/课时字段)作为新题库 `二建继续教育`。

## 目标

1. 进入 App 时先看到题库列表，点击题库后才进入「新建练习/历史练习」界面
2. 现有 141 题归入 `中级工程师`，历史记录保留并归到这个题库
3. 新导入 375 题作为 `二建继续教育`，支持按章节筛选练习
4. 切换题库后，历史记录互不干扰

## 非目标

- 不支持题库的增删改（题库定义写死在 `questions.js`）
- 不在错题回顾/答题页显示题库归属
- 章节内不再细分课时筛选（课时字段只存数据，UI 不展示）

## 数据结构

### `questions.js`

```js
const BANKS = [

      // ... 共 375 题

```

### 题目 id 作用域

题目 id 在题库内从 1 递增即可（不要求全局唯一）。一次练习里的所有题目都来自同一题库，`record.userAnswers[q.id]` 不会跨题库冲突。

### 判断题选项规范

判断题统一输出 `[{key:'正确',text:'正确'}, {key:'错误',text:'错误'}]`，答案是 `'正确'` 或 `'错误'`。与现有数据保持一致。

## 存储层

`localStorage["practice_records"]` 结构基本不变，每条 record 新增 `bankId` 字段：

```js
{
  id: '1747...',
  bankId: 'mid-engineer',   // 新增
  createdAt: '2026-05-19 10:30',
  finishedAt: null,
  status: 'in_progress',
  config: { types: [...], count: 20, random: true, chapters: ['第1章：施工技术'] },
  // ... 其余字段不变
  questions: [...],   // 题目快照（含章节字段，方便日后扩展）
  userAnswers: {...},
  ...
}
```

### 一次性迁移

在 App 启动时（`loadRecords()` 或 `renderBankList()` 第一次调用），扫描所有 record：若 `bankId` 字段缺失，写入 `'mid-engineer'`，然后写回 localStorage。下次启动就不会重复迁移。

### config.chapters

新加的 `config.chapters` 是一个字符串数组（章节名），仅在 `二建继续教育` 题库的练习中存在；旧题库的 record 没有这个字段。

## 业务逻辑改动

### 全局状态

新增 `currentBankId`（在 `bank-list` 视图被清空，在 `home` 视图被设置）。

### `renderBankList()`

```js
function renderBankList() {
  const listEl = document.getElementById('bank-list');
  listEl.innerHTML = BANKS.map(b => `
    <div class="bank-item" data-id="${b.id}">
      <div class="bank-info">
        <div class="bank-name">📚 ${b.name}</div>
        <div class="bank-count">${b.questions.length} 题</div>
      </div>
      <div class="bank-arrow">›</div>
    </div>
  `).join('');
  listEl.querySelectorAll('.bank-item').forEach(el => {
    el.addEventListener('click', () => enterBank(el.dataset.id));
  });
}

function enterBank(bankId) {
  currentBankId = bankId;
  showView('home');
  renderHome();
}

function getCurrentBank() {
  return BANKS.find(b => b.id === currentBankId);
}
```

### `buildPractice()` 改动

```js
function buildPractice(bank, config) {
  const { types, count, random, chapters } = config;
  let filtered = bank.questions.filter(q => types.includes(q.type));
  if (bank.hasChapters && chapters && chapters.length > 0) {
    filtered = filtered.filter(q => chapters.includes(q.chapter));
  }
  if (random) filtered = shuffleArray(filtered);
  return filtered.slice(0, count);
}
```

### `createPractice()` 改动

```js
function createPractice(config) {
  const bank = getCurrentBank();
  const questions = buildPractice(bank, config);
  return {
    id: String(Date.now()),
    bankId: currentBankId,   // 新增
    createdAt: nowString(),
    // ... 其余字段不变
  };
}
```

### `createReviewPractice()` 改动

复用 `sourceRecord.bankId`，保证错题重练也归到原题库。

### `updateNewPracticeUI()` 改动

不再读全局 `QUESTIONS`，改读 `getCurrentBank().questions`。同时根据 `currentBank.hasChapters` 显隐章节区，并渲染 chapter checkboxes。

### `getMaxCount()` 改动

考虑章节过滤：

```js
function getMaxCount() {
  const types = getSelectedTypes();
  if (!types.length) return 0;
  const bank = getCurrentBank();
  let filtered = bank.questions.filter(q => types.includes(q.type));
  if (bank.hasChapters) {
    const chapters = getSelectedChapters();
    if (chapters.length === 0) return 0;
    filtered = filtered.filter(q => chapters.includes(q.chapter));
  }
  return filtered.length;
}
```

### 一次性迁移

```js
function migrateRecords() {
  const records = loadRecords();
  let dirty = false;
  for (const r of records) {
    if (!r.bankId) {
      r.bankId = 'mid-engineer';
      dirty = true;
    }
  }
  if (dirty) saveRecords(records);
}
```

在 `DOMContentLoaded` 里调用一次，在 `renderBankList()` 之前。

## 数据导入

### 修改 `scripts/excel_to_json.py`

脚本变成「双输入 + 单输出」：

- 输入 1：`装饰工程网上学习题库.xlsx`（5 列，选项按 `|` 分割）→ 输出 bank `mid-engineer`
- 输入 2：`课后习题汇总.xlsx`（6 列：章节, 课时名称, 题型, 题目, 选项, 正确答案，选项按 `\n` 分割）→ 输出 bank `erjian-edu`
- 输出：`questions.js` 含 `const BANKS = [...]`

输出格式：

```js
const BANKS = [
  { id: 'mid-engineer', name: '中级工程师', hasChapters: false, questions: [...] },
  { id: 'erjian-edu', name: '二建继续教育', hasChapters: true, chapters: [...], questions: [...] }
];
```

### 解析细节

**选项解析**：
- 老 xlsx：按 `|` 分割
- 新 xlsx：按 `\n` 分割
- 都用同样的正则 `^([A-Z])[\.．、]\s*(.+)$` 抽取 key 和 text

**判断题处理**：
- 选项统一输出 `[{key:'正确',text:'正确'},{key:'错误',text:'错误'}]`，忽略 xlsx 里的选项列
- 答案是 `'正确'` 或 `'错误'`

**章节顺序**：
- `bank.chapters` 按 xlsx 中章节首次出现的顺序排列

## 缓存

`sw.js` 的 `CACHE_NAME` 从 `'deco-quiz-v2'` 升到 `'deco-quiz-v3'`，让用户刷新拉到新版。

## 错误处理 / 边界

| 场景                             | 处理                                                                       |
| -------------------------------- | -------------------------------------------------------------------------- |
| BANKS 为空                       | bank-list 显示「暂无题库」                                                 |
| 历史 record 的 bankId 不在 BANKS | 在该题库 home 不出现；考虑到不会发生，不做特殊提示                          |
| 题型一个不选                     | 「开始练习」alert「请至少选择一种题型」（保持原有逻辑）                     |
| 章节一个不选                     | 「开始练习」alert「请至少选择一个章节」（新增）                             |
| 章节全选且题型全选时题数 = 0     | maxCount = 0，滑块禁用（极端情况下兜底）                                    |
